# Platform VRT: Android

## Overview

Android exports require a pre-built APK. `runner/platform_runner.gd` acts as the
main scene; it reads `user://vrt_config.json` from the app's private storage to
determine which scenes to capture. Screenshots are saved to `user://vr_screenshots/`
and pulled from the emulator via `adb`.

---

## Architecture

```text
[Build phase]
  Godot Editor (or CI)
    └─ Export as Android → exports/android/godot_vrt_tests.apk
         (requires Android SDK + keystore configured)

[Capture phase — runs on Android Emulator or real device]

  adb push vrt_config.json /sdcard/Android/data/<pkg>/files/vrt_config.json
    │   (specifies which scenes to capture)
    │
  adb install godot_vrt_tests.apk
  adb shell am start -n <pkg>/.GodotApp
    │
    ├─ Godot runtime loads runner.tscn
    │    └─ platform_runner.gd reads user://vrt_config.json
    │
    ├─ Runner instantiates each scene in SubViewport (1280×720)
    │    └─ Processes .stories.json / .vrt.gd scripts
    │
    ├─ Screenshots → user://vr_screenshots/*.png
    │    (resolves to /sdcard/Android/data/<pkg>/files/vr_screenshots/)
    │
    └─ Writes user://vrt_done when finished
    │
  CI polls for user://vrt_done file via adb
  adb pull /sdcard/Android/data/<pkg>/files/vr_screenshots/ ./tests/vr_screenshots/
```

## Directory Layout

```text
godot-vrt/
├── runner/
│   └── platform_runner.gd   # Addon-level runner template
├── tests/
│   ├── runner/
│   │   ├── platform_runner.gd   # Copy inside the Godot project
│   │   └── runner.tscn          # Main scene set in export preset
│   ├── android_ui_test.gd           # Android-specific UI scene
│   ├── android_ui_test.tscn
│   ├── android_ui_test.stories.json
│   ├── android_ui_test_capture.vrt.gd
│   └── export_presets.cfg           # Android export preset (preset.1)
└── exports/
    └── android/                     # Build output (gitignored)
        └── godot_vrt_tests.apk
```

## Config File Format

Place `vrt_config.json` at `user://vrt_config.json` before launching the app.
On Android this resolves to `/sdcard/Android/data/<package>/files/vrt_config.json`.

```json
{
  "scenes": [
    "res://android_ui_test.tscn",
    "res://button_test.tscn",
    "res://click_test.tscn"
  ]
}
```

Omit `scenes` or leave the file absent to capture **all** `.tscn` files.

## Adding Android-Specific Scenes

1. Create `tests/your_scene.gd` (extends `Control`, `custom_minimum_size = Vector2(1280, 720)`)
2. Create `tests/your_scene.tscn`
3. Optionally create `tests/your_scene.stories.json` and `tests/your_scene_capture.vrt.gd`

**Android-specific patterns to test:**

| Pattern | Example |
| --- | --- |
| Bottom Navigation Bar | `android_ui_test.gd` — nav switching |
| Floating Action Button (FAB) | `android_ui_test.gd` — badge display |
| Status bar (time, icons) | `android_ui_test.gd` — status area |
| Material Design 3 cards | Elevation, rounded corners, ripple |
| Chip groups | Filter chips with active/inactive states |

## Local Verification

### Prerequisites

```bash
# Android SDK (install via Android Studio or sdkmanager)
# Set ANDROID_HOME environment variable
export ANDROID_HOME=$HOME/Android/Sdk
export PATH=$PATH:$ANDROID_HOME/emulator:$ANDROID_HOME/platform-tools

# Godot Android export template + Android build tools
# Configure in: Editor → Export → Android → (install preset)
```

### Step 1 — Start Android Emulator

```bash
# List available AVDs
emulator -list-avds

# Start emulator (replace 'Pixel_6_API_34' with your AVD name)
emulator -avd Pixel_6_API_34 -no-window &

# Wait for emulator to be ready
adb wait-for-device shell 'while [[ -z $(getprop sys.boot_completed) ]]; do sleep 1; done'
echo "Emulator ready"
```

### Step 2 — Export the APK

```bash
godot --headless --path tests/ --export-debug Android "$PWD/exports/android/godot_vrt_tests.apk"
```

> **Note:** Debug exports use the debug keystore, which is auto-generated by Godot.
> Release exports require a signing keystore configured in `export_presets.cfg`.

### Step 3 — Install APK and Push Config

```bash
PKG="com.example.godot_vrt_tests"
FILES_DIR="/sdcard/Android/data/${PKG}/files"

adb install -r exports/android/godot_vrt_tests.apk

# Create config (all scenes)
adb shell mkdir -p "${FILES_DIR}"
# To capture specific scenes only, push a config file:
# echo '{"scenes":["res://android_ui_test.tscn"]}' | \
#   adb shell "cat > ${FILES_DIR}/vrt_config.json"
```

### Step 4 — Launch and Wait

```bash
adb shell am start -n "${PKG}/.GodotApp"

# Wait for vrt_done (poll every 2 seconds, timeout 120s)
DONE_PATH="${FILES_DIR}/vrt_done"
for i in $(seq 1 60); do
  if adb shell "[ -f ${DONE_PATH} ] && echo yes" | grep -q yes; then
    echo "VRT complete."
    break
  fi
  sleep 2
done
```

### Step 5 — Pull Screenshots

```bash
mkdir -p tests/vr_screenshots
adb pull "${FILES_DIR}/vr_screenshots/." tests/vr_screenshots/
```

### Step 6 — Compare with Argos / pixelmatch

```bash
npm exec -- argos upload --token $ARGOS_TOKEN ./tests/vr_screenshots
```

## Troubleshooting

**INSTALL\_FAILED\_UPDATE\_INCOMPATIBLE**
: Uninstall the previous APK first: `adb uninstall com.example.godot_vrt_tests`

**App crashes on launch**
: Check `adb logcat | grep -i godot` for errors. Common cause is missing
export template or wrong architecture (arm64-v8a is required for most emulators).

**vrt\_done file never appears**
: The runner timed out or crashed. Use `adb logcat` to check Godot logs.
Ensure `WRITE_EXTERNAL_STORAGE` permission is enabled in `export_presets.cfg`.

**Screenshots are black**
: OpenGL ES renderer may not be fully initialized. Increase `SETTLE_FRAMES`
in `platform_runner.gd` (default: 5).
